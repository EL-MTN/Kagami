# Configuration

All env vars are declared in `apps/dashboard/src/env.ts` (the `@kagami/env`
spec — schema + docs in one place) and parsed once at module load by
`src/lib/cockpit.ts`. Cockpit is read-only and fail-open: every var is
defaulted or optional, and a bad URL value degrades to the Portless default
(with a console warning) instead of breaking the page.

There is no required configuration — with no `.env` at all the dashboard
works against the standard Portless topology. `KAO_TOKEN` is the only var
worth setting: it unlocks per-grant status on the Kao card (without it the
card shows "grant visibility not configured").

`.env` lives at `apps/dashboard/.env`. The table below,
`apps/dashboard/.env.example`, and `apps/dashboard/turbo.json` are
**generated** from the spec — edit `src/env.ts` and run `npm run env:gen`;
`npm run env:check` fails on drift.
